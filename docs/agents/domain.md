# 领域文档 (Domain Docs)

本文件指导工程 AI 技能在探索代码库时，应当如何阅读本项目的领域文档。

## 在修改代码前，请先阅读以下文档：

- **项目根目录的 `CONTEXT.md`**
- **`docs/adr/`** — 阅读与你即将工作领域相关的架构决策记录 (ADR)。

如果这些文件不存在，**请静默继续**。不要标记它们的缺失，也不要主动建议创建它们。相关技能（如 `/grill-with-docs`）会在业务术语或架构决策真正确定时，再进行延迟创建。

## 文件结构

本项目采用单上下文（Single-context）结构：

```text
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-event-sourced-orders.md
│   └── 0002-postgres-for-write-model.md
└── src/
```

## 严格使用词汇表中的术语

当你的输出涉及领域概念时（比如在 Issue 标题、重构建议、假设或测试用例名称中），请严格使用 `CONTEXT.md` 中定义的术语。切勿使用被词汇表明确弃用的同义词。

如果你需要的概念尚未在词汇表中出现，这通常是一个信号——要么你正在创造项目中原本不使用的术语（请重新考虑），要么确实存在术语空缺（请记下来，以便稍后通过 `/grill-with-docs` 补充）。

## 标记 ADR 冲突

如果你的修改计划或输出内容与现有的 ADR（架构决策记录）相矛盾，请明确指出，而不是默默覆盖：

> _这与 ADR-0007 (使用事件溯源订单) 相冲突 — 但值得重新讨论，因为……_
